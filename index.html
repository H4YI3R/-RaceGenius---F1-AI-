🏁 대표 프로젝트: RaceGenius

F1 레이스 전략 최적화를 위한 실시간 생성형 AI 어드바이저

저의 문제 해결 과정과 풀스택 역량을 보여주는 핵심 프로젝트입니다. 이 섹션에서는 프로젝트의 배경, 아키텍처, 정량적인 성과, 학습 내용을 상세히 설명합니다.

1. 프로젝트 개요 및 문제 정의

배경 (Background)

F1 레이스는 0.001초로 승부가 갈립니다. 현재 팀들은 '몬테카를로 시뮬레이션'을 통해 확률적으로 가장 높은 전략을 '예측'하지만, 수많은 실시간 변수(사고, 타이어 상태, 상대방의 예상치 못한 움직임) 속에서 '창의적인' 승부수를 찾는 데 한계가 있습니다.

문제 정의 (Problem)

"인간 전략가는 모든 변수를 실시간으로 고려할 수 없다." 기존 시뮬레이션은 정해진 시나리오 내에서의 '확률적 예측'에 불과합니다. 돌발 변수가 발생했을 때, 인간이 상상하지 못한 '창의적인' 대체 전략을 실시간으로 '생성'할 필요가 있습니다.

2. 해결 과정: 아키텍처

Transformer(Seq2Seq) 모델을 활용해 승리로 이어지는 다양한 '다음 행동' 시퀀스를 생성하도록 했습니다. 아래는 핵심 데이터 파이프라인입니다.

4. 배운 점 (Key Learnings)

기술적 성장: 생성형 AI(Transformer)를 F1 시계열 데이터에 적용하고 튜닝하며, Python 백엔드에서 AI 모델 결과를 React 프론트엔드에 실시간(WebSocket)으로 스트리밍하고 시각화하는 풀스택 역량을 확보했습니다.

시스템 아키텍처의 중요성: 실시간 F1 데이터를 받아 AI 모델을 거쳐 프론트엔드까지 끊김 없이(Low Latency) 전달하는 데이터 파이프라인을 설계하며, 안정적이고 확장성 있는 시스템 설계의 필요성을 깨달았습니다.
